After the FEMAP conversion is complete, the converted VTK result files can be visualized with the Field Plot tool.

This workflow is intended for viewing the field data generated from converted `.pvd` output, typically the file created by the FEMAP converter such as `.pyemsi/output.pvd`. The dialog reads field-file choices, array names, and range data from FEMAP conversion metadata cached in the current workspace.

See [Convert FEMAP Files](/docs/basics/FemapConverter.md) for the conversion step that produces the VTK output used here.

<img src="/pyemsi/img/field-plot-dialog.png" alt="field-plot-dialog" width="700" />

## Open The Field Plot Dialog

Open the field-plot builder from:

- `File -> Field Plot`

The dialog lets you choose a converted field file, configure how the result should be displayed, and open the field view inside pyemsi.


Or click on <img src="/pyemsi/img/Field.svg" alt="Run icon" width="20"/> icon on the main toolbar.

## Basic Workflow

1. Run the FEMAP conversion so the VTK output files are available.
2. Open the Field Plot dialog.
3. Select a cached converted field file from `Field File`.
4. Enable and configure one or more plot stages such as scalar, contour, vector, and optional feature-edge display.
5. If you are using vectors, click `Suggest` beside `Factor` when you want pyemsi to compute a glyph scale from the cached mesh size and array ranges.
6. Click `Plot` to open the field view in pyemsi.

## Main Inputs

The top of the dialog contains the main file selection fields.

| Field | Description |
| --- | --- |
| `Field File` | A combo box of cached converted field files available in the current workspace. In most cases this is the `.pvd` file generated by the FEMAP converter. The dialog defaults to the last selected cached field file when available. If the list is empty, rerun FEMAP conversion in the workspace first. |
| `Title` | The title used for the generated field-view tab in pyemsi. |

## Cached Field Data

The dialog reloads the cached field-file list each time it opens.

When you choose a cached `Field File`, pyemsi automatically:

- loads the available scalar names into the `Scalar` and `Contour` sections
- loads the available vector names into the `Vector` section
- updates vector `Scale` choices from the cached scalar and vector names

This data comes from FEMAP conversion metadata cached in the current workspace rather than from reading the selected file inside the dialog.

## Plot Stages

The dialog is organized into collapsible plotting stages. You can enable only the stages you need.

### Scalar

The `Scalar` section colors the mesh using a scalar field.

It includes:

- `Name`: the scalar field to display
- `Mode`: whether the values are shown from node data or element data
- `Colormap`: the colormap used to color the field
- `Show Edges`: toggles edge rendering on top of the scalar surface
- `Edge Color`: edge color used when `Show Edges` is enabled
- `Edge Opacity`: edge opacity used when `Show Edges` is enabled

### Contour

The `Contour` section draws contour lines from a selected scalar field.

It includes:

- `Name`: the scalar field used for contour generation
- `Contours`: the number of contour levels
- `Color`: the contour-line color
- `Line Width`: the contour-line thickness

### Vector

The `Vector` section displays vector quantities as glyphs.

It includes:

- `Name`: the vector or vector-related field to visualize
- `Scale`: how glyph scaling is determined
- `Glyph Type`: the glyph shape, such as arrow, cone, or sphere
- `Factor`: the multiplier applied to glyph size, with a `Suggest` button to compute a value from the cached mesh size and array ranges
- `Tolerance`: an optional tolerance control for glyph generation
- `Color Mode`: how glyph coloring is determined

The vector stage is especially useful for magnetic flux density, current density, and force vectors.

### Feature Edges

The `Feature Edges` section controls the mesh-edge overlay.

It includes:

- `Color`: feature-edge color
- `Line Width`: feature-edge thickness
- `Opacity`: feature-edge opacity
- `Remove Small Loops`: removes short closed edge loops from the extracted result
- `Edge Count Threshold`: the maximum loop size (in edge count) removed when `Remove Small Loops` is enabled
- `Feature Angle`: angle threshold passed to PyVista edge extraction; see [PyVista `extract_feature_edges` docs](https://docs.pyvista.org/api/core/_autosummary/pyvista.datasetfilters.extract_feature_edges)

This helps make geometry boundaries easier to read in the final field view.

## Script And Plot Actions

The bottom row of the dialog provides the main actions.

| Button | Description |
| --- | --- |
| `Script...` | Generates a Python script showing the equivalent `Plotter` commands for the current dialog configuration. |
| `Plot` | Builds the field plot and opens it in pyemsi. |
| `Cancel` | Closes the dialog without creating a plot. |

The `Suggest` button in the `Vector` section is the only inline helper action. It uses cached mesh and range metadata for the selected field file to suggest a vector `Factor`.

`Plot` and `Script...` require a cached `Field File` selection. Any enabled scalar, contour, or vector stage must also have a valid field selection.

## Result

When you click `Plot`, pyemsi creates a `Plotter` instance, applies the selected stages, and opens the resulting field view inside the GUI.

This allows you to inspect converted FEMAP results interactively after each conversion run.

For the underlying plotting API, see [Plotter](/docs/plotter/index.md).
